# Deploying PARKER to Vercel

The front end is a Vite + React app in `web/` with serverless API routes in
`web/api/`. Real data is generated by `web/scripts/build_data.py` and committed
as static JSON under `web/public/data/` (no secrets — just public football
data). The only secret the deployed site needs is the Anthropic key, used
server-side by the Ask Parker route.

## One-time setup (you do this)

1. **Push the branch** to GitHub (already committed on `parker-redesign`).

2. **Create the Vercel project**
   - Go to <https://vercel.com/new> and import this repository.
   - **Root Directory:** set to `web`  ← important, the app lives in `web/`, not the repo root.
   - Framework preset: **Vite** (auto-detected). Build command `npm run build`,
     output `dist` (already set in `web/vercel.json`).

3. **Add the environment variable** (Project → Settings → Environment Variables)

   | Name                | Value                    | Used by                         |
   | ------------------- | ------------------------ | ------------------------------- |
   | `ANTHROPIC_API_KEY` | your Anthropic API key   | `web/api/ask.js` (Ask Parker)   |

   That is the only variable required at runtime. Without it the Ask Parker bar
   still works but returns a graceful "engine unreachable" message instead of a
   live answer. `STATS_API_KEY` is **not** needed on Vercel — it is only used at
   data-generation time (below), never by the deployed site.

4. **Deploy.** Vercel builds the site and exposes `web/api/ask.js` as a
   serverless function at `/api/ask`. Done.

## Refreshing the live data

The committed JSON is a snapshot from the last generator run. To refresh it with
the latest results, standings and model predictions:

```bash
# from the repo root, with the World Cup API key available
STATS_API_KEY=<key> python web/scripts/build_data.py
git add web/public/data && git commit -m "Refresh data" && git push
```

Vercel auto-deploys on push, so the site updates. (The key is read from the env
var, or from `.streamlit/secrets.toml` for local runs — it never enters the repo
or the browser.)

For hands-off freshness, schedule the command above as a GitHub Action on a cron
(e.g. every few hours) that commits the refreshed JSON; each commit triggers a
Vercel redeploy. This keeps the Python model out of the request path (it only
runs at generation time) while the site stays current.

## Local development

```bash
cd web
npm install
npm run dev        # http://localhost:5173
```

The Vite dev server also serves `/api/*` (via a small middleware in
`vite.config.js`), so Ask Parker works locally too — export `ANTHROPIC_API_KEY`
in your shell before `npm run dev` to get live answers.
